- Test command execution through builder pattern's public API (String, Run, DryRun, Prompt)

- Test bubbletea TUI models by testing their constituent logic (categorization, selection) separately — not the model itself
- Use function-type fields in opts structs to inject TUI/scanner/adopter dependencies for testing orchestration flows
